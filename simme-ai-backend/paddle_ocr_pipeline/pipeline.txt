PIPELINE OCR (PaddleOCR) — Certificados de Calibração (CATIM)
=============================================================

Objectivo
---------
Processar PDFs de certificados de calibração e extrair informação estruturada
para posterior visualização, validação e integração numa interface.

Nesta fase, o pipeline já suporta:
- extração OCR por PDF
- segmentação por secções
- parsing de campos principais
- parsing de tabelas para alguns tipos de instrumento
- backend FastAPI para upload, processamento e consulta de resultados

Tipos de documento já testados
------------------------------
- Paquímetro / Laboratório de Metrologia - Dimensional
- Manómetro analógico / Laboratório de Metrologia - Pressões

Resumo das fases
----------------

01_pdf_to_images.py
- Lê os PDFs em data/raw_pdfs/
- Converte cada página em PNG
- Guarda em data/images/<nome_pdf>/page_XX.png

02_ocr_paddle.py
- Corre PaddleOCR sobre as imagens geradas
- Guarda o OCR bruto em data/ocr_text/<nome_pdf>.txt
- O TXT fica separado por páginas:
  === page_01.png ===
  ...
  === page_02.png ===

03_section_split.py
- Lê os ficheiros OCR em TXT
- Divide o conteúdo por páginas e secções semânticas
- Gera ficheiros em data/sections/
- Exemplo de secções:
  - header_meta
  - customer
  - equipment
  - work_conditions
  - description
  - traceability
  - uncertainty
  - results
  - observations

04_parse_fields.py
- Lê os ficheiros de secções
- Extrai campos estruturados para JSON
- Gera ficheiros em data/parsed/
- Estrutura típica:
  - header
  - customer
  - equipment
  - work_conditions
  - reference
  - raw_blocks

05_parse_tables.py
- Lê os ficheiros de secções e/ou parsed
- Identifica o tipo de instrumento
- Extrai tabelas específicas por tipo
- Gera ficheiros em data/tables/

Actualmente já suporta:
- caliper
  - E_contact_partial
  - S_scale_change
  - L_line_contact

- pressure_gauge
  - pressure_error_table
  - environmental_conditions
  - max_hysteresis (quando detectado correctamente)

06_metrics_phase1.py
- Calcula métricas simples por documento
- Pode servir para controlo de qualidade e debugging
- Dependendo da implementação, pode gerar resumos de completude ou consistência

07_gold_eval.py
- Compara resultados do pipeline com ficheiros gold anotados manualmente
- Gera métricas de avaliação
- Útil para a tese e validação científica

Backend FastAPI
---------------
Já existe um backend funcional em:
  backend/

Endpoints actuais:
- GET /health
- POST /upload
- POST /process/{file_id}
- GET /result/{file_id}

Fluxo:
1) Upload do PDF
2) Backend guarda o ficheiro em backend/uploads/
3) Backend copia o PDF para data/raw_pdfs/
4) Corre o pipeline completo (01 a 06)
5) Guarda o resultado final em backend/outputs/{file_id}.json
6) O frontend poderá depois consumir esse resultado

Compatibilidade importante
--------------------------
Este projecto está estabilizado para Windows com:
- Python 3.10
- numpy 1.26.4
- opencv-python 4.6.0.66
- opencv-contrib-python 4.6.0.66
- paddleocr 2.7.0.3

Se mudares estas versões, é comum aparecerem erros como:
- numpy ABI mismatch
- falha ao importar cv2
- conflitos com paddleocr

Como correr o pipeline manualmente
----------------------------------

1) Colocar PDFs em:
   data/raw_pdfs/

2) Converter PDFs em imagens:
   python .\src\01_pdf_to_images.py

3) OCR:
   python .\src\02_ocr_paddle.py

4) Segmentar secções:
   python .\src\03_segment_sections.py

5) Extrair campos:
   python .\src\04_parse_fields.py

6) Extrair tabelas:
   python .\src\05_parse_tables.py

7) Gerar métricas:
   python .\src\06_metrics_phase1.py

8) Avaliação com ground truth (opcional):
   python .\src\07_gold_eval.py

Dicas de troubleshooting
------------------------
Se o OCR falhar com numpy/cv2:
  python -c "import numpy, cv2; print(numpy.__version__, cv2.__version__)"

Esperado:
  numpy 1.26.4
  cv2 4.6.0

Se o 01_pdf_to_images.py falhar:
- normalmente é problema com Poppler
- confirmar se o Poppler está no PATH

Se o backend arrancar mas / der Not Found:
- é normal
- usar /docs ou /health
